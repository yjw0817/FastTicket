## 3. 주의사항

### Java/Spring 버전에 따른 차이

| 항목 | 현재 (eGov 3.10) | eGov 4.x |
|------|-------------------|----------|
| JSTL | `javax.servlet.jsp.jstl` | `jakarta.servlet.jsp.jstl` (Boot 3) |
| `<form:*>` 태그 | `spring-form.tld` | 동일 (Spring 5) / 변경 (Spring 6) |
| EL 표현식 | `${result.NAME}` | 동일 |
| `<ui:pagination>` | eGov 태그 | 버전 확인 필요 |

### eGov 4.x에서 확인할 것

1. **`<ui:pagination>` 출력 HTML** — flat 구조(`<a>` + `<strong>`)가 동일한지 확인
   - 다르면 `table.css`의 `#paging` 섹션 + `header.jsp`의 JS 수정 필요
2. **MyBatis `egovMap`** — 컬럼명이 여전히 대문자인지 확인
   - 다르면 List JSP의 EL 표현식 수정 필요
3. **Spring Form 태그** — `<form:select>`, `<form:input>` 네임스페이스 변경 여부
4. **Validator** — `validator.xml`의 message key 참조 방식 동일한지 확인

## 4. 빠른 마이그레이션 순서

```
1단계: CSS 폴더 통째로 복사
       css/egovframework/*.css (7개 파일)

2단계: header.jsp 복사
       (사이드바 + 탑바 + Bootstrap CDN + JS)

3단계: 각 JSP 페이지 복사 (16개)
       List 8개 + Register 8개

4단계: message-common.properties 병합
       기존 키 유지 + 신규 키 추가

5단계: 동작 확인
       - 페이지네이션 HTML 구조 확인
       - EL 표현식 대소문자 확인
       - JSTL/Spring Form 태그 정상 여부
```

## 5. Git 활용 팁

현재 커스터마이징 변경분만 패치로 추출:

```bash
# 전체 변경분을 패치 파일로 저장
git diff f63922b..HEAD > fastticket-customization.patch

# 새 프로젝트에서 적용 시도 (구조가 비슷할 경우)
git apply --check fastticket-customization.patch
git apply fastticket-customization.patch
```

> 프로젝트 구조가 달라졌다면 수동으로 파일별 복사가 더 안전합니다.
