# 암호화 가이드

## 개요

FastTicket은 개인정보 보호를 위해 비밀번호와 전화번호를 암호화하여 저장한다.

| 항목 | 방식 | 용도 |
|------|------|------|
| 비밀번호 | BCrypt (단방향) | 로그인 시 비교만 가능, 복호화 불가 |
| 전화번호 | AES-256-GCM (양방향) | 상세 조회 시 복호화, 목록에서는 뒷4자리만 표시 |
| 전화번호 뒷4자리 | 평문 (`TEL_LAST4`) | 목록 표시 및 검색용 |

## 핵심 클래스

### `com.ftk.common.util.CryptoUtil`

```java
// 비밀번호 해싱
CryptoUtil.hashPassword("1234")       // → "$2a$10$..."
CryptoUtil.matchPassword("1234", hash) // → true/false

// 전화번호 암호화/복호화
CryptoUtil.encrypt("010-1234-5678")    // → Base64 암호문
CryptoUtil.decrypt(encrypted)          // → "010-1234-5678"

// 뒷4자리 추출
CryptoUtil.extractLast4("010-1234-5678") // → "5678"
```

## DB 컬럼 구조

```sql
MEMBER 테이블:
  PASSWORD    VARCHAR(200)  -- BCrypt 해시 (60자이나 여유 확보)
  TEL         VARCHAR(500)  -- AES-256-GCM 암호문 (Base64)
  TEL_LAST4   VARCHAR(4)    -- 뒷4자리 평문 (조회/검색용)
```

## 데이터 흐름

### 회원 등록/수정 (INSERT/UPDATE)

```
Controller → ServiceImpl → DB
                │
                ├── password → CryptoUtil.hashPassword() → BCrypt 해시 저장
                ├── tel      → CryptoUtil.extractLast4() → TEL_LAST4에 평문 저장
                └── tel      → CryptoUtil.encrypt()      → TEL에 암호문 저장
```

### 회원 조회 (SELECT)

```
DB → ServiceImpl → Controller/JSP
        │
        └── tel → CryptoUtil.decrypt() → 평문 전화번호 반환
```

### 회원 목록 (SELECT LIST)

```
DB → ServiceImpl → JSP
        │
        └── TEL_LAST4만 조회 (복호화 불필요)
            JSP에서 "****-5678" 형태로 표시
```

### 로그인

```
1. memberId + USE_YN='Y' 로 DB 조회 (비밀번호 비교 없이)
2. ServiceImpl에서 CryptoUtil.matchPassword(입력값, DB해시) 비교
3. 일치하면 로그인 성공
```

## 기존 데이터 마이그레이션

### `com.ftk.common.init.PasswordMigrationRunner`

기존 평문 비밀번호/전화번호를 일괄 암호화하는 1회성 마이그레이션 클래스.

- 비밀번호: `$2a$`로 시작하지 않으면 BCrypt 해싱
- 전화번호: `0`으로 시작하는 평문이면 AES 암호화 + `TEL_LAST4` 채움
- SQL에서 미암호화 건만 조회하므로 이미 암호화된 데이터는 건너뜀

### 사용 방법

**평소에는 `@Component`가 주석 처리되어 있어 실행되지 않는다.**

마이그레이션이 필요할 때만 아래 절차를 따른다:

```
1. PasswordMigrationRunner.java 열기
2. // @Component  ← 주석 해제하여 @Component로 변경
3. Eclipse에서 빌드 → Tomcat 재시작
4. 로그에서 "암호화 마이그레이션 완료" 확인
5. 다시 @Component를 주석 처리 → // @Component
6. 빌드 → 재시작
```

### 마이그레이션이 필요한 경우
- DB에 직접 INSERT한 평문 데이터가 있을 때
- 새 환경에 기존 백업 데이터를 복원했을 때
- 신규 회원을 SQL로 대량 등록했을 때

## AES 키 관리

AES 키는 소스 코드가 아닌 별도 프로퍼티 파일에서 관리한다.

| 파일 | 용도 |
|------|------|
| `src/main/resources/egovframework/crypto.properties` | AES 키 저장 |
| `context-properties.xml` | 프로퍼티 로드 및 CryptoUtil Bean 등록 |

### crypto.properties
```properties
crypto.aes.key=FtkAes256SecretKey2026!@#$%^&*()
```

### 주의사항
- **Git에 커밋 금지** — `.gitignore`에 `crypto.properties` 추가 권장
- **운영/개발 환경별로 다른 키 사용** — 파일만 교체하면 됨
- **키 변경 시 기존 데이터 복호화 불가** — 전체 데이터 재암호화 마이그레이션 필요

## 배포 시 DB 변경

기존 테이블에 아래 ALTER 실행 필요:

```sql
-- TEL_LAST4 컬럼 추가
ALTER TABLE MEMBER ADD TEL_LAST4 VARCHAR(4);

-- PASSWORD 컬럼 확장 (BCrypt 해시 저장용)
ALTER TABLE MEMBER ALTER COLUMN PASSWORD VARCHAR(200);

-- TEL 컬럼 확장 (AES 암호화 저장용)
ALTER TABLE MEMBER ALTER COLUMN TEL VARCHAR(500);
```

`mssql-schema.sql`에 IF NOT EXISTS 조건과 함께 포함되어 있으므로, 스키마 파일 전체 실행으로도 적용 가능하다.

## 검색 기능

회원 목록에서 전화번호 뒷4자리로 검색 가능:

- 검색 조건: `searchCondition == 2` → `TEL_LAST4 LIKE '%keyword%'`
- 암호화된 TEL 컬럼은 검색 불가 (의도적 설계)
