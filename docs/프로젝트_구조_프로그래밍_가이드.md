# FastTicket 프로젝트 구조 및 프로그래밍 가이드

## 1. 개요

FastTicket은 전자정부 표준프레임워크(eGov 4.3.0) 기반의 티켓 예매 관리 시스템입니다.
WPMS 프로젝트(`com.dne.wpms`)의 패키지 구조를 참고하여, **업무 모듈별 + 레이어별 하위 패키지** 구조로 정리되어 있습니다.

> **기술 스택:** Spring 5.3.37, MyBatis, JSP, HSQLDB(기본), Tomcat 9.x (WAR 배포)

---

## 2. Java 패키지 구조

```
src/main/java/com/ftk/
│
├── booking/                    ← 예매
│   ├── controller/             ← @Controller (*.do 요청 처리)
│   │   └── BookingController.java
│   ├── mapper/                 ← @Mapper (MyBatis 인터페이스)
│   │   ├── BookingMapper.java
│   │   └── BookingDetailMapper.java
│   ├── service/                ← Service 인터페이스 + ServiceImpl
│   │   ├── BookingService.java
│   │   └── BookingServiceImpl.java
│   └── vo/                     ← VO (Value Object)
│       ├── BookingVO.java
│       └── BookingDetailVO.java
│
├── member/                     ← 회원 (동일 패턴)
├── price/                      ← 티켓 가격
├── program/                    ← 프로그램(공연)
├── sample/                     ← 샘플 (eGov 레퍼런스 CRUD)
├── schedule/                   ← 스케줄
├── seat/                       ← 좌석
├── venue/                      ← 공연장
│
├── common/                     ← 공통
│   ├── exception/              ← 예외 핸들러
│   │   ├── EgovSampleExcepHndlr.java
│   │   └── EgovSampleOthersExcepHndlr.java
│   └── vo/                     ← 공통 VO
│       └── CommonDefaultVO.java
│
└── config/                     ← 설정
    └── web/                    ← 웹 설정
        ├── EgovBindingInitializer.java
        └── EgovImgPaginationRenderer.java
```

### 2.1 모듈별 패키지 규칙

각 업무 모듈은 아래 4개 하위 패키지를 가집니다:

| 패키지 | 역할 | 네이밍 규칙 |
|--------|------|-------------|
| `controller/` | HTTP 요청 처리 | `{Entity}Controller.java` |
| `mapper/` | MyBatis DB 인터페이스 | `{Entity}Mapper.java` (@Mapper) |
| `service/` | 비즈니스 로직 인터페이스 + 구현체 | `{Entity}Service.java` (인터페이스), `{Entity}ServiceImpl.java` |
| `vo/` | 데이터 전달 객체 | `{Entity}VO.java` |

### 2.2 새 모듈 추가 시

예: `event`(이벤트) 모듈 추가

```
com/ftk/event/
├── controller/EventController.java
├── mapper/EventMapper.java
├── service/EventService.java
├── service/EventServiceImpl.java
└── vo/EventVO.java
```

**체크리스트:**
1. VO는 `CommonDefaultVO` 또는 `SampleDefaultVO`를 상속 (페이지네이션/검색 지원)
2. Mapper 인터페이스에 `@Mapper` 어노테이션 필수
3. ServiceImpl은 `EgovAbstractServiceImpl` 상속
4. ID 생성이 필요하면 `context-idgen.xml`에 빈 등록
5. `sql-mapper-config.xml`에 typeAlias 등록
6. `resources/mapper/{모듈}/` 에 SQL XML 생성

## 3. Resources 구조

```
src/main/resources/
│
├── mapper/                              ← MyBatis SQL XML (모듈별 분리)
│   ├── booking/
│   │   ├── Booking_SQL.xml
│   │   └── BookingDetail_SQL.xml
│   ├── member/Member_SQL.xml
│   ├── price/TicketPrice_SQL.xml
│   ├── program/Program_SQL.xml
│   ├── sample/EgovSample_Sample_SQL.xml
│   ├── schedule/Schedule_SQL.xml
│   ├── seat/Seat_SQL.xml
│   └── venue/Venue_SQL.xml
│
├── egovframework/
│   ├── spring/                          ← Spring XML 설정
│   │   ├── context-aspect.xml           ← AOP (예외 전파)
│   │   ├── context-common.xml           ← 컴포넌트 스캔
│   │   ├── context-datasource.xml       ← DB 연결 (HSQLDB/MySQL/Oracle)
│   │   ├── context-idgen.xml            ← ID 생성 서비스 빈
│   │   ├── context-mapper.xml           ← MyBatis SqlSession, Mapper 스캔
│   │   ├── context-properties.xml       ← 앱 속성 (pageUnit, pageSize)
│   │   ├── context-transaction.xml      ← 트랜잭션 AOP
│   │   └── context-validator.xml        ← Validation 설정
│   │
│   ├── sqlmap/example/
│   │   └── sql-mapper-config.xml        ← MyBatis typeAlias 정의
│   │
│   └── message/                         ← i18n 메시지 번들
│       ├── message-common.properties    ← 기본
│       ├── message-common_ko.properties ← 한국어
│       └── message-common_en.properties ← 영어
│
├── db/                                  ← DB 스키마 초기화
│   ├── sampledb.sql                     ← HSQLDB DDL
│   └── mssql-schema.sql                 ← MSSQL DDL
│
└── log4j2.xml                           ← 로깅 설정
```

### 3.1 Java Mapper ↔ SQL XML 매핑

Java의 `@Mapper` 인터페이스와 SQL XML은 **namespace로 1:1 매핑**됩니다:

| Java Mapper | SQL XML namespace |
|-------------|-------------------|
| `com.ftk.booking.mapper.BookingMapper` | `mapper/booking/Booking_SQL.xml` |
| `com.ftk.member.mapper.MemberMapper` | `mapper/member/Member_SQL.xml` |
| `com.ftk.sample.mapper.SampleMapper` | `mapper/sample/EgovSample_Sample_SQL.xml` |

SQL XML의 `<mapper namespace="...">` 값이 Java Mapper의 풀 패키지 경로와 일치해야 합니다.

## 5. Spring XML 설정 요약

| 파일 | 핵심 설정 |
|------|----------|
| `context-common.xml` | `<context:component-scan base-package="com.ftk">` |
| `context-mapper.xml` | `mapperLocations = classpath:/mapper/**/*.xml` |
| `context-aspect.xml` | `pointcut = execution(* com.ftk..service.*Impl.*(..))` |
| `context-transaction.xml` | 동일 pointcut, `DataSourceTransactionManager` |
| `dispatcher-servlet.xml` | 뷰 prefix: `/WEB-INF/jsp/ftk/`, 컨트롤러 스캔 |

> **참고:** 컴포넌트 스캔과 Mapper 스캔 모두 `com.ftk` 베이스이므로,
> 하위 패키지를 추가/변경해도 스캔 설정은 수정할 필요가 없습니다.

## 6. 아키텍처 흐름

```
[클라이언트] → *.do URL
    ↓
[Controller] (com.ftk.{mod}.controller)
    ↓ Service 인터페이스 호출
[ServiceImpl] (com.ftk.{mod}.service)
    ↓ Mapper 인터페이스 호출
[Mapper] (com.ftk.{mod}.mapper)
    ↓ namespace 매핑
[SQL XML] (resources/mapper/{mod}/)
    ↓
[Database]
```

### 6.1 주요 패턴

- **PRG 패턴:** POST 처리 후 `redirect:` 로 GET 리다이렉트
- **ID 생성:** `EgovIdGnrService`로 테이블 기반 채번 (예: `BOOKING-00001`)
- **페이지네이션:** `SampleDefaultVO` 상속 → `PaginationInfo` + `EgovImgPaginationRenderer`
- **트랜잭션:** AOP 기반, `*Impl` 클래스 메서드에 자동 적용
- **DI:** `@Resource` (eGov 기본) 또는 `@RequiredArgsConstructor` (Lombok)

## 7. WPMS와의 구조 비교

FastTicket은 WPMS(`com.dne.wpms`)의 패키지 구조를 참고하여 설계되었습니다.

| 구분 | WPMS | FastTicket |
|------|------|------------|
| 프레임워크 | Spring Boot | eGov 4.3 (Spring MVC) |
| 패키지 구조 | `{대분류}/{업무}/controller,mapper,service,vo` | `{모듈}/controller,mapper,service,vo` |
| SQL XML | `resources/mapper/{대분류}/{업무}/` | `resources/mapper/{모듈}/` |
| 공통 | `common/` (util, vo, exception, handler) | `common/` (exception, vo) |
| 설정 | `config/` (db, security, web) | `config/` (web) |
| 앱 설정 | `application.properties` | Spring XML (`context-*.xml`) |
| 서블릿 | 내장 톰캣 (Spring Boot) | 외장 Tomcat 9 + `web.xml` |
