# 프로그램별 권종/할인 적용 개선

## 작성일: 2026-03-10

## 목표
1. **프로그램별 권종 매핑** — PROGRAM_TICKET_TYPE 테이블로 프로그램에 사용할 권종 선택
2. **할인 프로그램별 override** — DISCOUNT_PROGRAM.DISCOUNT_VALUE로 프로그램별 할인값 재정의
3. **할인 반올림 설정** — DISCOUNT에 ROUNDING_UNIT, ROUNDING_TYPE 추가
4. **DiscountCalcUtil** — 할인 계산 + 반올림 정적 유틸

## 구현 순서

### Step 1: DB 스키마 변경
- PROGRAM_TICKET_TYPE 테이블 신설
- TICKET_TYPE.TYPE_NM UNIQUE 제약조건
- DISCOUNT_PROGRAM.DISCOUNT_VALUE 컬럼 추가
- DISCOUNT.ROUNDING_UNIT, ROUNDING_TYPE 컬럼 추가

### Step 2: TICKET_TYPE 중복 체크 쿼리
- TicketType_SQL.xml에 selectTicketTypeByName 추가

### Step 3: PROGRAM_TICKET_TYPE (VO → Mapper → SQL → Service → Controller → JSP)
- ProgramVO.typeIds 필드 추가
- ProgramMapper에 insert/delete/select 쿼리
- ProgramServiceImpl에서 저장/조회 로직
- ProgramController에서 ticketTypeList 전달
- programRegister.jsp에 체크박스 UI

### Step 4: 가격 설정 권종 필터링
- SessionPrice_SQL.xml: PROGRAM_TICKET_TYPE 조인 필터 (비어있으면 전체)
- ProductPrice_SQL.xml: 동일

### Step 5: 할인 반올림 (VO → SQL → DiscountCalcUtil)
- DiscountVO에 roundingUnit, roundingType 필드
- Discount_SQL.xml INSERT/UPDATE/SELECT 반영
- DiscountCalcUtil 신규 클래스

### Step 6: 할인 프로그램별 override (VO → SQL → Service → JSP)
- DiscountVO에 programValues (List<Integer>) 필드
- DISCOUNT_PROGRAM INSERT에 DISCOUNT_VALUE 추가
- DiscountServiceImpl.saveDiscountPrograms 수정
- discountList.jsp에 인라인 할인값 + 반올림 UI

### Step 7: 온라인예약 할인 적용
- OnlineBookingController에서 DiscountCalcUtil 연동
