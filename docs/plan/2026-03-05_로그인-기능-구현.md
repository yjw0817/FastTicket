# 로그인 기능 구현 계획

## Context
FastTicket에 로그인/로그아웃 기능을 추가한다. 현재 모든 페이지가 인증 없이 접근 가능하며, header.jsp에 "관리자"가 하드코딩되어 있다. Spring Security 없이 세션 기반으로 구현하며, 로그아웃 전까지 세션이 유지되어야 한다.

## 기존 자산
- MEMBER 테이블: `sampledb.sql`에 이미 정의됨 (MEMBER_ID, PASSWORD 컬럼 포함)
- MemberVO: password 필드 이미 존재
- MemberMapper/Service/ServiceImpl: CRUD 메서드 구현 완료
- 테스트용 데이터만 sampledb.sql에 INSERT 추가 필요

## 구현 상세

### 1. Member_SQL.xml — 로그인 쿼리
기존 `resultMap id="member"`를 재사용한다.
```xml
<select id="selectMemberByLogin" parameterType="memberVO" resultMap="member">
    SELECT MEMBER_ID, MEMBER_NM, PASSWORD, TEL, EMAIL, USE_YN, REG_DT
    FROM MEMBER
    WHERE MEMBER_ID = #{memberId}
      AND PASSWORD = #{password}
      AND USE_YN = 'Y'
</select>
```

### 2. MemberMapper → MemberService → MemberServiceImpl
```java
MemberVO selectMemberByLogin(MemberVO vo) throws Exception;
```
- 결과 null이면 null 반환 (예외 아님 — Controller에서 에러 메시지 처리)

### 3. LoginController
- `GET /login.do` → login.jsp 표시
- `POST /login.do` → ID/PW 검증 → 성공: 세션 저장 + redirect → 실패: 에러 메시지
- `GET /logout.do` → 세션 무효화 + redirect:/login.do
- 세션 키: `loginVO` (MemberVO 객체)
- `@RequiredArgsConstructor` 사용

### 4. LoginInterceptor
- `preHandle()`에서 `session.getAttribute("loginVO")` 확인
- 미인증 → `/login.do`로 redirect
- 제외 URL: `/login.do`, `/logout.do`

### 5. dispatcher-servlet.xml
```xml
<mvc:interceptors>
    <mvc:interceptor>
        <mvc:mapping path="/**/*.do"/>
        <mvc:exclude-mapping path="/login.do"/>
        <mvc:exclude-mapping path="/logout.do"/>
        <bean class="com.ftk.common.interceptor.LoginInterceptor"/>
    </mvc:interceptor>
</mvc:interceptors>
```

### 6. login.jsp
- 독립 페이지 (header.jsp 미포함, sidebar 없음)
- 중앙 정렬 카드 형태, Bootstrap 5 CDN
- 필드: 회원ID, 비밀번호 + 에러 메시지 영역

### 7. header.jsp
- `관리자` 하드코딩 → `${sessionScope.loginVO.memberNm}` 표시
- 아바타: 이름 첫 글자
- 로그아웃 버튼 추가

### 8. sampledb.sql
```sql
INSERT INTO MEMBER VALUES('admin','관리자','1234','010-0000-0000','admin@ftk.com','Y',NOW());
```

## 비밀번호 보안
현재 단계: 평문 저장. BCrypt 해싱은 Spring Security 도입 시 함께 적용.

## 검증
1. 임의 페이지 접근 → `/login.do`로 redirect
2. `admin` / `1234` 로그인 → 메인 이동 + 헤더에 "관리자" 표시
3. 로그아웃 → 세션 무효화 + 로그인 페이지 이동
4. 로그아웃 후 URL 직접 입력 → 로그인 페이지로 redirect
