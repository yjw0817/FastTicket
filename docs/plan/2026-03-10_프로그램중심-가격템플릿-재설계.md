# 프로그램 중심 가격 템플릿 시스템 재설계

## Context

현재 시스템의 문제:
- 권종(TICKET_TYPE)이 글로벌 마스터 → 프로그램과 분리되어 관리 복잡
- 할인(DISCOUNT)도 글로벌 → 경로할인이 어린이에 적용되는 등 오적용 발생
- 회차(SCHEDULE) + 가격(SESSION_PRICE)을 날짜마다 전부 생성 → 데이터 폭발
- 가격 변경 시 수백 개 행을 수동 수정해야 함

**목표**: 설계.md 원안대로 프로그램 중심 + 템플릿 기반 + 예외만 저장

## 핵심 원칙

1. **프로그램이 중심**: 권종+기본가격, 회차설정, 할인 모두 프로그램 안에서 관리
2. **템플릿이 기본값**: 평일/주말/공휴일 × 회차 × 권종 가격 템플릿
3. **예외만 저장**: 변동 있는 날짜만 override 저장 + 사유 필수
4. **회차 제어**: 템플릿에서도, 특정 날짜에서도 enable/disable (사유 입력)

## 데이터 모델

### 1. PROGRAM 테이블 확장

```sql
ALTER TABLE PROGRAM ADD SESSION_COUNT INT NULL;           -- 회차 수
ALTER TABLE PROGRAM ADD FIRST_SESSION_TIME VARCHAR(5) NULL; -- 첫 회차 시작 (HH:mm)
ALTER TABLE PROGRAM ADD SESSION_INTERVAL INT NULL;         -- 회차 간격(분), 공연시간+휴식 포함
ALTER TABLE PROGRAM ADD DEFAULT_ONLINE_CAP INT NULL DEFAULT 0;
ALTER TABLE PROGRAM ADD DEFAULT_OFFLINE_CAP INT NULL DEFAULT 0;
```

### 2. PROGRAM_TICKET_TYPE 변경 (기본가격 추가)

```sql
ALTER TABLE PROGRAM_TICKET_TYPE ADD BASE_PRICE INT DEFAULT 0;  -- 평일 기본가격
ALTER TABLE PROGRAM_TICKET_TYPE ADD SORT_ORDER INT DEFAULT 0;
```
- TICKET_TYPE 마스터는 **autocomplete 소스로만** 사용 (별도 메뉴 유지)
- 실제 가격은 PROGRAM_TICKET_TYPE.BASE_PRICE가 기준

### 3. SESSION_TEMPLATE (회차 템플릿 — 신규)

```sql
CREATE TABLE SESSION_TEMPLATE (
    PROGRAM_ID  VARCHAR(16) NOT NULL,
    SESSION_NO  INT NOT NULL,
    DAY_TYPE    VARCHAR(10) NOT NULL,  -- WEEKDAY / WEEKEND / HOLIDAY
    EVENT_TIME  VARCHAR(5) NOT NULL,   -- HH:mm
    ENABLED     CHAR(1) DEFAULT 'Y',
    PRIMARY KEY (PROGRAM_ID, SESSION_NO, DAY_TYPE)
);
```
- 프로그램 회차 설정 시 자동 생성 (3 DAY_TYPE × N 회차)
- 주말에 3회차 운영 안하면 → 해당 행 ENABLED='N'

### 4. PRICE_TEMPLATE (가격 템플릿 — 신규)

```sql
CREATE TABLE PRICE_TEMPLATE (
    PROGRAM_ID  VARCHAR(16) NOT NULL,
    SESSION_NO  INT NOT NULL,
    TYPE_ID     VARCHAR(16) NOT NULL,
    DAY_TYPE    VARCHAR(10) NOT NULL,  -- WEEKDAY / WEEKEND / HOLIDAY
    PRICE       INT DEFAULT 0,
    PRIMARY KEY (PROGRAM_ID, SESSION_NO, TYPE_ID, DAY_TYPE)
);
```
- WEEKDAY 가격 = PROGRAM_TICKET_TYPE.BASE_PRICE 초기값
- WEEKEND/HOLIDAY 가격은 별도 설정 (일괄 인상 기능)

### 5. DATE_PRICE_OVERRIDE (날짜별 가격 변동 — 신규)

```sql
CREATE TABLE DATE_PRICE_OVERRIDE (
    PROGRAM_ID  VARCHAR(16) NOT NULL,
    EVENT_DATE  DATE NOT NULL,
    SESSION_NO  INT NOT NULL,
    TYPE_ID     VARCHAR(16) NOT NULL,
    PRICE       INT NOT NULL,
    REASON      VARCHAR(200) NOT NULL,  -- 변동 사유 필수
    REG_DT      DATETIME DEFAULT GETDATE(),
    PRIMARY KEY (PROGRAM_ID, EVENT_DATE, SESSION_NO, TYPE_ID)
);
```

### 6. DATE_SESSION_OVERRIDE (날짜별 회차 on/off — 신규)

```sql
CREATE TABLE DATE_SESSION_OVERRIDE (
    PROGRAM_ID  VARCHAR(16) NOT NULL,
    EVENT_DATE  DATE NOT NULL,
    SESSION_NO  INT NOT NULL,
    ENABLED     CHAR(1) NOT NULL,       -- Y(강제활성) / N(비활성)
    REASON      VARCHAR(200) NOT NULL,  -- 사유 필수
    REG_DT      DATETIME DEFAULT GETDATE(),
    PRIMARY KEY (PROGRAM_ID, EVENT_DATE, SESSION_NO)
);
```

### 7. PROGRAM_DISCOUNT (프로그램별 할인 — 신규)

```sql
CREATE TABLE PROGRAM_DISCOUNT (
    PD_ID           VARCHAR(16) NOT NULL PRIMARY KEY,
    PROGRAM_ID      VARCHAR(16) NOT NULL,
    DISCOUNT_NM     VARCHAR(100) NOT NULL,
    DISCOUNT_TYPE   VARCHAR(10) NOT NULL,   -- PERCENT / AMOUNT
    DISCOUNT_VALUE  INT NOT NULL,
    VERIFY_REQUIRED CHAR(1) DEFAULT 'N',   -- 확인필요 (신분증 등)
    ROUNDING_UNIT   INT DEFAULT 1,
    ROUNDING_TYPE   VARCHAR(10) DEFAULT 'ROUND',
    USE_YN          CHAR(1) DEFAULT 'Y',
    SORT_ORDER      INT DEFAULT 0,
    REG_DT          DATETIME DEFAULT GETDATE()
);
```
- 기존 DISCOUNT 테이블 → "공통 할인 템플릿" 역할 (가져오기용)
- PROGRAM_DISCOUNT = 프로그램에 실제 적용되는 할인
- 중복 적용 불가, 최대 할인 자동 적용
- 확인필요: 온라인 예약 시 "현장 확인 필요" 표시

### 8. SCHEDULE 테이블 (유지)

- 기존 구조 유지 — 예약 용량 관리(좌석 차감) 용도
- 생성 방식 변경: 템플릿 기반 자동 생성 (기간 선택 → 생성)
- **SESSION_PRICE 테이블은 폐지** → PRICE_TEMPLATE + DATE_PRICE_OVERRIDE로 대체

### 가격 조회 로직 (런타임)

```
getPrice(programId, eventDate, sessionNo, typeId):
  1. DATE_PRICE_OVERRIDE에서 조회
     → 있으면 override 가격 반환
  2. 없으면 DAY_TYPE 판단:
     - HOLIDAY 테이블에 있으면 → 'HOLIDAY'
     - 토/일 → 'WEEKEND'
     - 그 외 → 'WEEKDAY'
  3. PRICE_TEMPLATE에서 (programId, sessionNo, typeId, dayType) 조회
```

## 메뉴 구조 변경

### Before (현재)
```
장소/시설 → 회차입장권(프로그램/일정/회차별가격) → 일반입장권 → 좌석 → 예매 → 회원
→ 시스템관리 → 기초설정(권종관리/휴무일/영업시간/할인정책)
```

### After (개편)
```
장소/시설 → 입장권(프로그램 목록) → 좌석 → 예매 → 회원
→ 시스템관리 → 기초설정(휴무일/영업시간)
```

- '일정'과 '회차별가격'이 프로그램 상세 안으로 통합
- '권종관리'는 autocomplete 소스로만 유지 (메뉴에서 제거 or 축소)
- '할인정책' 글로벌 화면은 "공통 할인 템플릿" 용도로 축소

## 온라인 예약 연동 변경

### 날짜 조회
```sql
-- 예약 가능 날짜: SCHEDULE이 존재하고, 활성 상태인 날짜
SELECT DISTINCT EVENT_DATE FROM SCHEDULE
WHERE PROGRAM_ID = ? AND STATUS = 'OPEN' AND ONLINE_AVAIL > 0
  AND EVENT_DATE > GETDATE()
  AND NOT EXISTS (
    SELECT 1 FROM DATE_SESSION_OVERRIDE DSO
    WHERE DSO.PROGRAM_ID = SCHEDULE.PROGRAM_ID
      AND DSO.EVENT_DATE = SCHEDULE.EVENT_DATE
      AND DSO.SESSION_NO = SCHEDULE.SESSION_NO
      AND DSO.ENABLED = 'N'
  )
```

### 가격 조회
```sql
-- 회차의 가격: override 우선, 없으면 template
SELECT TT.TYPE_ID, TT.TYPE_NM,
    COALESCE(DPO.PRICE, PT.PRICE) AS PRICE,
    CASE WHEN DPO.PRICE IS NOT NULL THEN DPO.REASON END AS PRICE_REASON
FROM PROGRAM_TICKET_TYPE PTT
JOIN TICKET_TYPE TT ON PTT.TYPE_ID = TT.TYPE_ID
LEFT JOIN PRICE_TEMPLATE PT
    ON PT.PROGRAM_ID = PTT.PROGRAM_ID
   AND PT.SESSION_NO = #{sessionNo}
   AND PT.TYPE_ID = PTT.TYPE_ID
   AND PT.DAY_TYPE = #{dayType}  -- 런타임 계산
LEFT JOIN DATE_PRICE_OVERRIDE DPO
    ON DPO.PROGRAM_ID = PTT.PROGRAM_ID
   AND DPO.EVENT_DATE = #{eventDate}
   AND DPO.SESSION_NO = #{sessionNo}
   AND DPO.TYPE_ID = PTT.TYPE_ID
WHERE PTT.PROGRAM_ID = #{programId}
ORDER BY PTT.SORT_ORDER
```

### 할인 조회
```sql
-- 기존 DISCOUNT → PROGRAM_DISCOUNT로 변경
SELECT * FROM PROGRAM_DISCOUNT
WHERE PROGRAM_ID = #{programId} AND USE_YN = 'Y'
```

## 구현 순서

### Phase 1: DB 스키마 + 프로그램 화면 개편
1. 신규 테이블 생성 (SESSION_TEMPLATE, PRICE_TEMPLATE, DATE_PRICE_OVERRIDE, DATE_SESSION_OVERRIDE, PROGRAM_DISCOUNT)
2. PROGRAM 테이블 컬럼 추가, PROGRAM_TICKET_TYPE에 BASE_PRICE/SORT_ORDER 추가
3. ProgramVO 필드 추가, 프로그램 등록 JSP 개편 (권종 인라인 + 회차설정 + 할인)
4. Program Controller/Service에 권종+기본가격+할인 저장 로직

### Phase 2: 가격 템플릿 시스템
5. 템플릿 VO/Mapper/SQL/Service 구현
6. 기존 '회차별 가격' 화면 → 탭 기반 템플릿 관리 UI로 개편
7. 템플릿 자동생성 로직 (권종+회차설정 입력 시 WEEKDAY 템플릿 생성)
8. 주말/공휴일 일괄 가격 설정 기능

### Phase 3: 달력 + 예외 관리
9. 기존 '일정' 화면 → 달력 뷰로 개편
10. 날짜별 가격 override CRUD + 사유 입력
11. 날짜별 회차 enable/disable + 사유 입력
12. 스케줄 생성 (템플릿 기반, 기간 선택)

### Phase 4: 온라인 예약 연동
13. 가격 조회 로직 변경 (SESSION_PRICE → PRICE_TEMPLATE + override)
14. 할인 조회 변경 (DISCOUNT → PROGRAM_DISCOUNT)
15. 확인필요 할인 UI 표시

### Phase 5: 정리
16. SESSION_PRICE 의존성 제거, 기존 데이터 마이그레이션
17. 메뉴 구조 정리
18. 기존 글로벌 할인 화면 → "공통 할인 템플릿" 용도로 축소

## 검증 방법

1. 프로그램 등록 시 권종+기본가격 인라인 추가 → 저장 후 재조회
2. 회차 설정 → 템플릿 자동생성 확인 (3 DAY_TYPE × N회차 × K권종)
3. 주말 탭에서 일괄 +1000원 → 저장 확인
4. 회차 비활성 → 저장 확인 (템플릿 수준)
5. 달력에서 특정 날짜 가격 변경 → override 저장 + 사유 확인
6. 특정 날짜 회차 비활성 → override 저장 + 사유 확인
7. 온라인 예약에서 PRICE_TEMPLATE → override 우선 적용 확인
8. PROGRAM_DISCOUNT 할인 적용 + 확인필요 표시 확인
9. HOLIDAY 등록된 날짜가 공휴일 가격으로 표시되는지 확인
