# 회의록 vs 현재 개발 — GAP 분석

## Context
2026.03.01 회의에서 논의된 FastTicket 핵심 비즈니스 요구사항과 현재 구현된 코드베이스 간의 차이점 분석.
회의 핵심: **회차 입장권이 프로젝트의 90% 포인트** — 범용 티켓 발권 시스템으로서의 유연성이 핵심.

---

## 회의에서 정의한 3가지 상품 유형

| 유형 | 설명 | 정원 체크 | 회차 | 우선순위 |
|------|------|-----------|------|----------|
| **상품** | 매점 물품 판매 (기존 시스템 카피) | X | X | 낮음 |
| **일반 입장권** | 단순 입장 (수목원 등 정원 없는 곳) | X | X | 중간 |
| **회차 입장권** | 시간대별 입장 (수영장, 케이블카 등) | O | O | **최고 (90%)** |

---

## GAP 분석: 현재 구현 vs 요구사항

### 1. 상품 유형 구분 없음 ⚠️ 핵심
- **현재**: `PROGRAM.PROGRAM_TYPE` = ADMISSION/PERFORMANCE/MUSICAL (공연 중심)
- **필요**: 상품 / 일반입장권 / 회차입장권을 **완전히 분리된 메뉴/UI**로 구성
- **회의 원문**: "상품 입장권 회차 입장권... 아예 분리를 할 거야"

### 2. 회차 자동 생성 없음 ⚠️ 핵심
- **현재**: `SCHEDULE` 테이블에 일정을 **수동으로 하나씩** 등록
- **필요**:
  - 영업시간(시작~종료) + 단위(60분 등) + 기간(1/1~12/31) 입력
  - → 1년치 회차가 **자동 생성** (365일 × N회차)
  - 인터벌(쉬는시간) 설정 가능
- **회의 원문**: "여기서 1월 1일부터 12월 31일까지 넣으면 1년 치 365일이 만들어져야 돼"

### 3. 정원(온라인/오프라인) 분리 없음 ⚠️
- **현재**: `SCHEDULE.TOTAL_SEATS` / `AVAIL_SEATS` — 단일 정원
- **필요**: 정원을 **온라인 정원 + 오프라인 정원** 2개로 분리
  - 온라인에서 미판매분 → 현장(오프라인)에 넘김
  - 당일 온라인 예약 차단 (전제조건)
- **회의 원문**: "오프라인 온라인 나눠서 할 거... 정원을 나눠"

### 4. 권종별 가격이 회차별로 다를 수 없음 ⚠️ 핵심
- **현재**: `TICKET_PRICE`가 **PROGRAM에 고정** — 모든 회차가 동일 가격
- **필요**:
  - 기초 설정에서 권종(어린이/청소년/성인 등) 등록
  - 기본 금액(디폴트) 설정 → 전 회차에 자동 적용
  - **회차별, 요일별로 금액 개별 수정** 가능
  - 회차 × 권종 = 가격 (다대다 관계)
- **회의 원문**: "2회차에 어린이는 천원 2천원 3천원... 거기다가 할인 들어가면..."

### 5. 요일별 가격 설정 없음
- **현재**: 요일 개념 없음
- **필요**: 주 단위로 기본 세팅 → 특별한 날만 개별 수정
- **회의 원문**: "화요일만 금액을 바꾸고 싶으면 화요일만 바꿀 수 있어"

### 6. 기초 설정(권종 관리) 없음
- **현재**: `PRICE_TYPE`이 코드로 고정 (ADULT/CHILD/INFANT/SENIOR/DISABLED)
- **필요**: 업체별로 커스텀 권종 등록 (유아/어린이/청소년/성인/경로/대학생 등 최대 ~6개)
- **회의 원문**: "유아 어린이 청소년 경로 대학생 성인 경로까지 딱 6개"

### 7. 할인 정책 없음 (후순위)
- **현재**: 미구현
- **필요**: 별도 할인 정책 (회의에서 "나중에"로 후순위 결정)
- **회의 원문**: "할인 정책은 특별한 케이스일 때만 바꾸는 걸로"

### 8. 일반 입장권 UI 분리 없음
- **현재**: 모든 것이 Program → Schedule 흐름
- **필요**: 일반 입장권은 회차/정원 없이 **권종 × 금액만** (별도 UI)
- **회의 원문**: "회차 입장권과 일반 입장권은 UI가 틀려... 아예 다른 놈"

### 9. 영업시간/휴무일 관리 없음
- **현재**: 미구현
- **필요**: 요일별 영업시간, 공휴일, 휴무일, 단축영업일 관리
- **회의 원문**: "공휴일 영업 시간도 따로 있고... 휴가일하고 다 체크해 놓은 거"

### 10. 용어 불일치 (일부 진행중)
- 공연장 → **장소/시설** (변경 시작함)
- 공연일정 → **회차** 또는 **시간대**
- 프로그램 → 상황에 따라 **상품/입장권/회차입장권**
- 테이블 헤더에 아직 "공연장", "공연일정" 등 잔존

### Phase 1: 기초 설정 & 데이터 모델 변경 ✅
1. **권종(기초 설정) 관리** — TICKET_TYPE 별도 테이블 ✅
2. **SCHEDULE 테이블 확장** — 온라인/오프라인 정원, 회차번호 ✅
3. **회차별 가격 테이블 신설** — SESSION_PRICE (SCHEDULE_ID × 권종 → 금액) ✅

### Phase 2: 회차 입장권 핵심 UI ✅
4. **회차 자동 생성 UI** — 기간 + 영업시간 + 단위 → 벌크 생성 ✅
5. **회차별 가격 일괄/개별 수정 UI** — 시간표 형태 그리드 ✅

### Phase 3: 상품 유형 분리 ✅
6. **메뉴 분리** — SESSION / ADMISSION / PRODUCT 타입 분리 ✅
7. **일반 입장권 별도 UI** — 회차 없는 단순 권종+금액 등록 ✅

### Phase 4: 부가 기능 ✅
8. 영업시간/휴무일 관리 — BUSINESS_HOURS + HOLIDAY + 벌크 생성 연동 ✅
9. 할인 정책 — DISCOUNT 기초 데이터 관리 ✅

### Phase 5: 온라인 예약 연동 ✅
10. 온라인 예약 — 고객용 5단계 위저드, 원자적 정원 차감, 당일 예약 차단 ✅
