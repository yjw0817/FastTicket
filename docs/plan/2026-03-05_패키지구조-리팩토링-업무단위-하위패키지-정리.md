# 패키지 구조 리팩토링: 업무 단위 하위 패키지 정리

**생성일:** 2026-03-05
**상태:** 계획 수립

## Context

WPMS 프로젝트(`com.dne.wpms`)의 구조를 참고하여, 각 업무 모듈을 `controller/mapper/service/vo`로 정리.
WPMS는 `common/` (공통 유틸/VO/예외), `config/` (설정) 패턴 사용.

## 변경 매핑

### 도메인 모듈 (booking, member, price, program, sample, schedule, seat, venue)

| 현재 | 목표 |
|------|------|
| `{mod}/web/*Controller` | `{mod}/controller/*Controller` |
| `{mod}/service/impl/*Mapper` | `{mod}/mapper/*Mapper` |
| `{mod}/service/impl/*ServiceImpl` | `{mod}/service/*ServiceImpl` (flatten) |
| `{mod}/service/*VO`, `*DefaultVO` | `{mod}/vo/*VO` |
| `{mod}/service/*Service` (인터페이스) | `{mod}/service/` (유지) |

### cmmn → common + config (WPMS 패턴 적용)

| 현재 | 목표 |
|------|------|
| `cmmn/EgovSampleExcepHndlr` | `common/exception/EgovSampleExcepHndlr` |
| `cmmn/EgovSampleOthersExcepHndlr` | `common/exception/EgovSampleOthersExcepHndlr` |
| `cmmn/service/CommonDefaultVO` | `common/vo/CommonDefaultVO` |
| `cmmn/web/EgovBindingInitializer` | `config/web/EgovBindingInitializer` |
| `cmmn/web/EgovImgPaginationRenderer` | `config/web/EgovImgPaginationRenderer` |

## 실행 순서

### 1단계: 파일 물리적 이동

각 도메인 모듈별로 `controller/`, `mapper/`, `vo/` 디렉토리를 생성하고 파일을 이동한다.

#### 1-1. 도메인 모듈 이동

각 모듈(`booking`, `member`, `price`, `program`, `sample`, `schedule`, `seat`, `venue`)에 대해:

```bash
# controller/ 생성 및 이동
mkdir -p {mod}/controller/
mv {mod}/web/*Controller.java {mod}/controller/

# mapper/ 생성 및 이동
mkdir -p {mod}/mapper/
mv {mod}/service/impl/*Mapper.java {mod}/mapper/

# ServiceImpl은 service/impl/ → service/ 로 한 단계 올림
mv {mod}/service/impl/*ServiceImpl.java {mod}/service/

# vo/ 생성 및 이동
mkdir -p {mod}/vo/
mv {mod}/service/*VO.java {mod}/vo/
mv {mod}/service/*DefaultVO.java {mod}/vo/  # (sample 모듈의 SampleDefaultVO 등)
```

#### 1-2. cmmn → common + config 이동

```bash
# common/exception/
mkdir -p common/exception/
mv cmmn/EgovSampleExcepHndlr.java common/exception/
mv cmmn/EgovSampleOthersExcepHndlr.java common/exception/

# common/vo/
mkdir -p common/vo/
mv cmmn/service/CommonDefaultVO.java common/vo/

# config/web/
mkdir -p config/web/
mv cmmn/web/EgovBindingInitializer.java config/web/
mv cmmn/web/EgovImgPaginationRenderer.java config/web/
```

#### 1-3. 빈 디렉토리 삭제

```bash
# 각 모듈의 빈 web/, service/impl/ 삭제
rmdir {mod}/web/
rmdir {mod}/service/impl/

# cmmn/ 전체 삭제 (빈 상태)
rm -rf cmmn/
```

### 2단계: Java 파일 package/import 일괄 수정

#### 2-1. package 선언 수정 (이동된 파일)

| 대상 파일 | 변경 전 | 변경 후 |
|-----------|---------|---------|
| `*Controller.java` | `package com.ftk.{mod}.web;` | `package com.ftk.{mod}.controller;` |
| `*Mapper.java` | `package com.ftk.{mod}.service.impl;` | `package com.ftk.{mod}.mapper;` |
| `*ServiceImpl.java` | `package com.ftk.{mod}.service.impl;` | `package com.ftk.{mod}.service;` |
| `*VO.java`, `*DefaultVO.java` | `package com.ftk.{mod}.service;` | `package com.ftk.{mod}.vo;` |
| `CommonDefaultVO.java` | `package com.ftk.cmmn.service;` | `package com.ftk.common.vo;` |
| `EgovSample*ExcepHndlr.java` | `package com.ftk.cmmn;` | `package com.ftk.common.exception;` |
| `EgovBindingInitializer.java` | `package com.ftk.cmmn.web;` | `package com.ftk.config.web;` |
| `EgovImgPaginationRenderer.java` | `package com.ftk.cmmn.web;` | `package com.ftk.config.web;` |

#### 2-2. import 문 수정 (전체 Java 파일 대상)

모든 Java 파일에서 아래 패턴을 일괄 치환:

```
# web → controller (8개 모듈 각각)
com.ftk.{mod}.web  →  com.ftk.{mod}.controller

# VO import 수정 (8개 모듈 각각)
com.ftk.{mod}.service.{X}VO  →  com.ftk.{mod}.vo.{X}VO
com.ftk.{mod}.service.{X}DefaultVO  →  com.ftk.{mod}.vo.{X}DefaultVO

# service.impl 분기 처리
com.ftk.{mod}.service.impl.{X}Mapper  →  com.ftk.{mod}.mapper.{X}Mapper
# (ServiceImpl은 service.impl → service 이므로 같은 패키지 내 import 불필요)

# cmmn 관련
com.ftk.cmmn.service.CommonDefaultVO  →  com.ftk.common.vo.CommonDefaultVO
com.ftk.cmmn.web  →  com.ftk.config.web
com.ftk.cmmn.EgovSampleExcepHndlr  →  com.ftk.common.exception.EgovSampleExcepHndlr
com.ftk.cmmn.EgovSampleOthersExcepHndlr  →  com.ftk.common.exception.EgovSampleOthersExcepHndlr
```

#### 2-3. ServiceImpl에서 Mapper import 추가

ServiceImpl이 `service/impl/` → `service/`로 이동하면서 Mapper와 같은 패키지가 아니게 됨.
각 ServiceImpl 파일에 `import com.ftk.{mod}.mapper.{X}Mapper;` 추가 필요.

### 3단계: Spring XML 설정 수정

#### 3-1. `context-aspect.xml`

```xml
<!-- pointcut 변경 -->
변경 전: execution(* com.ftk..impl.*Impl.*(..))
변경 후: execution(* com.ftk..service.*Impl.*(..))

<!-- bean class 변경 -->
변경 전: com.ftk.cmmn.EgovSampleExcepHndlr
변경 후: com.ftk.common.exception.EgovSampleExcepHndlr

변경 전: com.ftk.cmmn.EgovSampleOthersExcepHndlr
변경 후: com.ftk.common.exception.EgovSampleOthersExcepHndlr
```

#### 3-2. `context-transaction.xml`

```xml
<!-- pointcut 변경 -->
변경 전: execution(* com.ftk..impl.*Impl.*(..))
변경 후: execution(* com.ftk..service.*Impl.*(..))
```

#### 3-3. `dispatcher-servlet.xml`

```xml
변경 전: com.ftk.cmmn.web.EgovBindingInitializer

변경 전: com.ftk.cmmn.web.EgovImgPaginationRenderer
변경 후: com.ftk.config.web.EgovImgPaginationRenderer
```

### 4단계: MyBatis 설정 수정

#### 4-1. `sql-mapper-config.xml` (typeAlias)

각 모듈의 VO alias 변경:

```xml
<!-- 변경 전 -->
<typeAlias alias="sampleVO" type="com.ftk.sample.service.SampleVO"/>
<!-- 변경 후 -->
<typeAlias alias="sampleVO" type="com.ftk.sample.vo.SampleVO"/>

<!-- 모든 VO에 대해 동일 적용 -->
<!-- cmmn → common -->
변경 전: com.ftk.cmmn.service.CommonDefaultVO
변경 후: com.ftk.common.vo.CommonDefaultVO
```

#### 4-2. `*_SQL.xml` (namespace)

각 매퍼 XML의 namespace 변경:

```xml
<!-- 변경 전 -->
<mapper namespace="com.ftk.{mod}.service.impl.{X}Mapper">
<!-- 변경 후 -->
<mapper namespace="com.ftk.{mod}.mapper.{X}Mapper">
```

## 주의사항

- `SampleDefaultVO`는 모든 모듈의 VO가 상속하는 베이스 클래스 → `sample/vo/`로 이동 후 import 경로가 올바른지 반드시 확인
- `CommonDefaultVO`는 `SampleDefaultVO`의 부모 → 변경 순서에 따른 의존성 주의
- Spring component scan은 `com.ftk` 베이스이므로 하위 패키지 이동에 영향 없음
- MyBatis mapper scan도 `com.ftk` 베이스이므로 영향 없음
- `@Mapper` 어노테이션은 유지 (MyBatis mapper scan이 인식)
